This guide walks you through installing dependencies, registering a Discord bot, configuring the script, and running it for the first time.

## Prerequisites

- **Python 3.8+** installed and available as `python3`
- **pip** (bundled with Python 3)
- A **Discord account** with permission to manage a server
- One or more **FiveM servers** accessible from the machine running the bot

## Install dependencies

`json` and `datetime` are part of the Python standard library. You only need to install two third-party packages:

```bash
pip install discord.py requests
```

<Note>
If your system has both Python 2 and Python 3, use `pip3` instead of `pip`.
</Note>

## Get the script

Clone the repository or download `status.py` directly:

```bash
git clone https://github.com/jcfoxtrot54/FiveM-Discord-Bot.git
cd FiveM-Discord-Bot
```

## Register a Discord bot

<Steps>
  <Step title="Create a Discord application">
    Go to the [Discord Developer Portal](https://discord.com/developers/applications) and click **New Application**. Give it a name (e.g. `Status Bot`) and click **Create**.
  </Step>

  <Step title="Add a bot user">
    In the left sidebar, select **Bot**, then click **Add Bot** and confirm. Under the **Token** section, click **Reset Token** and copy the token — you will need it in the next section.

    <Note>
      Your bot token grants full control over the bot account. Never commit it to version control, share it publicly, or hard-code it in any file you distribute. Store it in an environment variable or a secrets manager for production use.
    </Note>
  </Step>

  <Step title="Invite the bot to your server">
    In the left sidebar, select **OAuth2 → URL Generator**. Under **Scopes**, check `bot`. Under **Bot Permissions**, check at minimum:

    - **Read Messages / View Channels**
    - **Send Messages**
    - **Manage Messages** (required for the channel purge)
    - **Embed Links**

    Copy the generated URL, open it in a browser, and authorise the bot to join your server.
  </Step>

  <Step title="Copy the channel ID">
    In Discord, enable **Settings → Advanced → Developer Mode**. Right-click the channel you want the bot to post in and click **Copy ID**. Save this number — it goes into `ChannelID` below.
  </Step>
</Steps>

## Configure status.py

Open `status.py` and edit the six variables at the top of the file.

### ChannelID

Paste the channel ID you copied above:

```python
ChannelID = 709801509210030160  # replace with your channel ID
```

### BotToken

Paste the token you copied from the Developer Portal:

```python
BotToken = "your-token-here"
```

<Warning>
Do not share or commit your `BotToken`. If it is ever exposed, immediately regenerate it in the Developer Portal.
</Warning>

### PublicIP

Set this to the public hostname or IP address players use to connect to your FiveM servers:

```python
PublicIP = "velocitynetworks.org"
```

This value is combined with each server's port to build the direct-connect string shown in the embed (e.g. `velocitynetworks.org:30120`).

### Servers

Define every server you want to monitor. Set `"Bypass": True` to insert an empty line break between groups instead of a server entry:

```python
Servers = [
    {"URL": "http://localhost:30120", "DisplayName": "Los Santos #1", "Bypass": False},
    {"URL": "http://localhost:30122", "DisplayName": "Los Santos #2", "Bypass": False},
    {"Bypass": True},  # blank line separator
    {"URL": "http://localhost:30124", "DisplayName": "Los Santos #3", "Bypass": False},
]
```

- **URL** — The internal HTTP address of the FiveM server. The bot calls `<URL>/players.json` and `<URL>/info.json`.
- **DisplayName** — The label shown as the embed field title.
- **Bypass** — When `True`, the entry renders as an invisible spacer; all other keys are ignored.

### EmbedTitle

The heading shown at the top of the status embed:

```python
EmbedTitle = "Velocity Networks FiveM Server Status"
```

### UpdateDelay

How many seconds the bot waits between each status refresh (minimum recommended: `5`):

```python
UpdateDelay = 5
```

<Tip>
Lowering `UpdateDelay` increases how quickly the embed reflects server changes, but also increases the number of HTTP requests made to your FiveM servers and to the Discord API. Values below `5` are not recommended.
</Tip>

## Run the bot

```bash
python3 status.py
```

## What to expect

On a successful startup you will see the following in your terminal:

```
Connected as YourBotName#1234!
```

The bot will then:

1. Purge all existing messages in the configured channel.
2. Post a `Loading...` placeholder message.
3. Replace the placeholder with a Discord embed listing every server's current player count and online status.
4. Edit that same embed every `UpdateDelay` seconds with fresh data.

After `CounterCap` updates (default `30`) the bot purges the channel and posts a fresh embed, preventing Discord from accumulating edited-message history.

A server that responds successfully displays its current and maximum player count along with a direct-connect address:

```
🟢  14/64 players
Find me on the FiveM server listings!
Direct connect via: `velocitynetworks.org:30120`
```

A server that does not respond within the 2-second timeout displays the down notice instead:

```
🔴 No response 😢

 I'm probably just offline
```
